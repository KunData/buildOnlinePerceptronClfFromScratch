Build Online Perceptron Classifier from Scratch for Restaurant Review Classification with N-gram Model¶

Kun Tao¶

I INTRODUCTION¶

This project aims to build an Online Perceptron classifier in Python from scratch for a binary restaurant review classification under a real-world scenario: efficiently train the classifier with one million restaurant reviews and ratings, then make highly accurate prediction on 320,122 test examples to find out whether or not the ratings of restaurants are at least 4 out of 5 stars based on customers' reviews.

As the first step of the classifier development, a n-gram model is used to represent the restaurant reviews, that is, each word or each group of words appears in the training dataset is treated as a feature, and its value for a specific document in the corpus is the counts of its occurrences. When setting n = 1, 2, or 3, I get the Unigram, Bigram or Trigram correspondingly, which, in terms of the training set, are respectively a 1,000,000 by 207,393, 1,000,000 by 6,680,976, or 1,000,000 by 32,145,597 sparse matrix after affine expansion. The Unigram only makes use of the information reflected by the term frequency of each individual word, while the Bigram or Trigram exploits the message revealed by n (n >= 2) adjacent words, more like human speech is supposed to be. Therefore, in this project, I adopt the Bigram model to train and test our self-developed Online Perceptron classifier. To deal with the challenges posted by the high dimensional Bigram feature matrix, I further explore the Mini-Batch online learning and the Chi-square test feature selection to achieve a good balance between the computational efficiency and the prediction accuracy.

This report is organized as follows: Section II introduces an implementation of the basic Online Perceptron algorithm, while Section III and IV respectively discuss the Mini-Batch Online Perceptron and the Online Perceptron with Feature Selection by Chi-sqaure test. A short summary and comparison can be found in Section V.

II BASIC IMPLEMENTATION OF ONLINE PERCEPTRON ALGORITHM¶

Online Perceptron algorithm performs stochastic gradient descent, which updates the weight vector immediately after every single wrong prediction during a pass over the randomly shuffled training dataset. The performance of Online Perceptron is sensitive to the order in which the training examples are fed. To guarantee the prediction accuracy, I traverse through the training data twice to train the classifier, and further carry out the online-to-batch conversion in the second pass by taking the average of the 1,000,001 weight vectors.

As shown from Line 17 to Line 61 in the following block, the implementation of Online Perceptron is pretty straightforward. However, its training speed turns out to be terribly slow in our case: on a computer with 2.7 GHz Intel Core i7 and 12 GB RAM, it took over 4 hours to obtain the final classifier of which the training error rate is 4.97% and the test error rate is 10.06%. The root cause of this time-consuming training process arises from the need to loop through the 1,000,000 by 6,680,976 feature matrix twice to execute operations required by Online Perceptron algorithm.

def onlinePerceptron(X, Y):
    
    YX = X.multiply(Y)
    YX = YX.tocsr()
    
    totalExs = X.shape[0]
    totalFeats = X.shape[1]
    w = np.zeros((1, totalFeats), dtype=np.float)

    for i in range(totalExs):
        wx = X[i].dot(w.T)
        
        if Y[i] * wx <= 0:
            w = w + YX[i]
            
    return w

def aveOnlinePerceptron(X, Y, w):
    u0 = w
    u = w
    c = 1
    
    totalExs = X.shape[0]
    rowIdx = np.arange(totalExs)
    np.random.shuffle(rowIdx)
    X = X[rowIdx, :]
    Y = Y[rowIdx, :]
    
    YX = X.multiply(Y)
    YX = YX.tocsr()
    
    for i in range(totalExs):
        wx = X[i].dot(w.T)
        
        if Y[i] * wx <= 0:
            w = w + YX[i]
            u = u + c * YX[i]
            
        c = c + 1
        
    w = u0/c + w - 1/c * u
    
    return w

#==================================================================================
def getErrorRate(X, Y, wFinal):
    
    totalEx = X.shape[0]
    preds = np.zeros((totalEx, 1)) 
    
    preds = X.dot(wFinal.T)
    preds[preds <= 0] = -1
    preds[preds > 0] = 1
    
    errors = np.sum(Y != preds)
    errorRate = float(errors / totalEx)
    
    return errorRate

III Mini-Batch ONLINE PERCEPTRON¶

One way to speed up Online Perceptron when handling high-dimensional data is to take advantage of matrix operations instead of simply looping through the training examples one by one. In other words, split the one million training data into M small batches so that each batch only contains N training examples (M * N = 1,000,000). Then for each mini-batch, apply matrix operations to calculate the dot product and update the weight vectors when needed.

def miniBatchOnlinePerceptron(X, Y, N, M):
    
    YX = X.multiply(Y)
    YX = YX.tocsr()
    
    totalFeats = X.shape[1]
    w = np.zeros((1, totalFeats), dtype=np.float)
    
    for i in range(M):
        x = X[i*N:(i+1)*N, :]
        y = Y[i*N:(i+1)*N, :]
        yx = YX[i*N:(i+1)*N, :]

        wx = x.dot(w.T)
        ywx = np.multiply(y, wx)
        idx = np.where(ywx <= 0.0)
        idx = idx[0]
        
        if(idx.shape[0] > 0):
            yx = yx[idx]
            yxSum = yx.sum(axis=0)
            yxAve = np.multiply(yxSum, 1/N)
            
            w = w + yxAve
            
    return w

def aveMiniBatchOnlinePerceptron(X, Y, w, N, M):
    u0 = w
    u = w
    c = 1
    
    totalExs = X.shape[0]
    totalFeats = X.shape[1]
    
    rowIdx = np.arange(totalExs)
    np.random.shuffle(rowIdx)
    X = X[rowIdx, :]
    Y = Y[rowIdx, :]
    
    YX = X.multiply(Y)
    YX = YX.tocsr()
    
    for i in range(M):
        x = X[i*N:(i+1)*N, :]
        y = Y[i*N:(i+1)*N, :]
        yx = YX[i*N:(i+1)*N, :]
        
        wx = x.dot(w.T)
        ywx = np.multiply(y, wx)
        idx = np.where(ywx <= 0.0)
        idx = idx[0]
        
        if(idx.shape[0] > 0):
            yx = yx[idx]
            yxSum = yx.sum(axis=0)
            yxAve = np.multiply(yxSum, 1/N)
            
            w = w + yxAve
            u = u + c * yxAve
            
        c = c + 1
        
    w = u0/c + w - 1/c * u
    
    return w

This Mini-Batch variation of Online Perceptron implementation has been proven to be significantly faster than the "naive" one. For instance, when setting 1000 training examples per batch, it only took about 7 miniutes to obtain the final weight vector, meanwhile, the test error rate is just 9.48%, which is even better than that got from the original Online Perceptron classifier. Varying the size of a mini-batch, the training time and error rates also change. In general, the bigger the mini-batch, the shorter the training time and the slightly higher the error rates.

In addition, checking the feature names which correspond to the top and the bottom 50 weights, it is easy to see that both groups are indeed a good indicator of their respective classes. In other words, when a customer review contains any words in the group of the top 50 feature names, he/she is very likely to rate the restaurant as 4 stars or above (correspondingly to lable +1), and vice versa.

IV ONLINE PERCEPTRON WITH FEATURE SELECTION BY CHI-SQUARE TEST¶

An alternative way to improve the computation efficiency of the original Online Perceptron is to reduce the dimensionality of the Bigram matrix by feature selection. The following code block shows how to use the sklearn SelectKBest class with a chi2 score function to do so.

def featureSelector4Training(Xtrain, Ytrain, feature_names):
    
    ch2_selector = SelectKBest(chi2, k="all")
    ch2_selector = ch2_selector.fit(Xtrain, Ytrain)
    ch2_selector_scores = ch2_selector.scores_
    ch2_selector_pvalues = ch2_selector.pvalues_
    ch2_selector_scores_max = np.max(ch2_selector_scores)
    ch2_selector_scores_min = np.min(ch2_selector_scores)
    ch2_selector_pvalues_max = np.max(ch2_selector_pvalues)
    ch2_selector_pvalues_min = np.min(ch2_selector_pvalues)
    print("ch2_scores_max = {}\tch2_scores_min = {}".format(ch2_selector_scores_max, \
                                                                         ch2_selector_scores_min))
    print("ch2_pvalues_min = {}\t\t\tch2_pvalues_max = {}\n".format(ch2_selector_pvalues_min, \
                                                                         ch2_selector_pvalues_max))

    ch2_selector_scores_idx_top50 = ch2_selector_scores.argsort()[-50:][::-1] 

    top50_feature_names = []
    for i in ch2_selector_scores_idx_top50:
        top50_feature_names.append(feature_names[i])   
    print("Top 50 feature names with highest chi-square test scores:\n{}\n".format(top50_feature_names))

    ch2_selector_scores_idx_bot50 = ch2_selector_scores.argsort()[:50]
    
    bot50_feature_names = []
    for i in ch2_selector_scores_idx_bot50:
        bot50_feature_names.append(feature_names[i])   
    print("Bot 50 feature names with lowest chi-square test scores:\n{}\n".format(bot50_feature_names))
    

    f1 = plt.figure(1)
    plt.hist(ch2_selector_scores, bins=100, range=(0, 100))
    plt.title("Lower End of Chi-square Scores Histogram")

    kk = (ch2_selector_pvalues < 1e-4).sum()
    ch2_selector = SelectKBest(chi2, k=kk)
    ch2_selector = ch2_selector.fit(Xtrain, Ytrain)
    Xtrain = ch2_selector.transform(Xtrain) 

    selected_feature_names = []
    selected_feature_scores = []
    selected_feature_pvalues = []
    if feature_names:
        for i in ch2_selector.get_support(indices=True):
            selected_feature_names.append(feature_names[i])
            selected_feature_scores.append(ch2_selector_scores[i])
            selected_feature_pvalues.append(ch2_selector_pvalues[i])
        
    selected_feature_scores = np.asarray(selected_feature_scores)
    selected_feature_pvalues = np.asarray(selected_feature_pvalues)

    f2 = plt.figure(2)
    plt.hist(selected_feature_scores, bins=100, range=(0, 100))
    plt.title("Lower End of Chi-square Selected Feature Scores Histogram")

    return Xtrain, ch2_selector

Feature selection based on the sklearn chi2 function computes the Chi-squared statistics to measure the independence between each feature and the class. The higher the calculated statistics, the lower the p-value, and the more UNLIKELY the feature is independent of the class label. Setting a threshold on the upper bound of the p-values, I get the most relevant n (n << 6,680,976) features to the classification, which will greatly accelerate the training process of Online Perceptron. For example, let ch2_selector_pvalues < 1e-4, the dimensionality of the feature matrix is reduced to 1,000,000 by 101040, consequently, the training time is shortened to about 5 minutes, and the test error rate (9.77%) is pretty good.

Compared with the Mini-Batch approach, although the Chi-square based feature selection does enhance the computational efficiency of the Online Perceptron classifier in our case, neither the feature names with the top and bottom 50 chi-square test scores, nor those with the highest and lowest 50 final weights offer a good intuitive explanation of why the selected or abandoned features should or should not work well for the classification.

V CONCLUSION¶

In this project, I'd like to demonstrate how to build an efficient and accurate Online Perceptron Classifier in Python from scratch for text classification. To handle the challenges brought by the 1,000,000$\times$6,680,976-dimensional training feature matrix, I further improve the original Online Perceptron implementation with two different approaches: one is the Mini-Batch Stochastic Gradient Descent which speeds up the time-consuming training process by batch-based matrix operations and much less iterations, the other is the Chi-square feature selection preprocessing which significantly reduces the training features from 1,000,000 to 101,040. Test error rates obtained from these two enhanced Online Perceptron classifiers are 9.48% and 9.77%, respectively, both beat the benchmark error rate of 10.06%. As for the training time, both methods only need about 5 ~ 7 minutes, which is 35 times faster than the original classifier.

In terms model interpretability, the Mini-Batch based classifier does much better than the Chi-square feature-selection one, and its final weight vector actually provides a good feature importance measure: weights with large absolute values correspond to very positive or negative comments, which naturally implies a +1 or -1 class label. On the contrary, the weight-to-feature mapping after the Chi-square based feature selection processing is intuitively hard to explain from the classification point of view.
